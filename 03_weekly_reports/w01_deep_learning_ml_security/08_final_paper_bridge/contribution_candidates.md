# Contribution 후보

1. 딥러닝/ML 시스템을 데이터 관리, 모델 학습, 검증, 배포·운영 단계로 나누고, 각 단계의 보안 위협과 평가 지표를 연결하는 생명주기 기반 평가 프레임워크를 제안한다.
2. Clean performance, robust performance, privacy leakage, reproducibility evidence를 분리해 기록하는 평가 프로토콜을 정리한다.
3. 침입탐지, 대적 ML, 프라이버시 공격 문헌을 CIA 및 Privacy/Safety/Accountability 관점으로 비교하는 문헌 매트릭스를 제공한다.
4. Docker, seed, config, DOI/URL 검증표, AI 활용 기록을 포함한 재현성 중심 보고서 작성 절차를 제시한다.
5. 악용 가능성이 있는 공격 실습을 실제 서비스 공격이 아닌 toy/public data 기반의 안전한 평가 설계로 제한하는 연구윤리 기준을 명시한다.

## 기여 후보 우선순위

| 우선순위 | Contribution | 이유 |
|---:|---|---|
| 1 | 생명주기 기반 평가 프레임워크 | 모든 주차 문헌을 하나의 구조로 묶을 수 있다. |
| 2 | 네 가지 평가축 분리 | 성능 중심 보고서의 한계를 직접 보완한다. |
| 3 | 재현성 중심 작성 절차 | 제출물 품질과 AI 활용 투명성을 강화한다. |
