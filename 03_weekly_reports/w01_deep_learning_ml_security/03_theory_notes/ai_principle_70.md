# AI 원리 70% 정리

## 1. 핵심 이론

W01의 AI 원리는 딥러닝을 보안 연구의 대상으로 보기 전에, 딥러닝이 어떤 방식으로 학습하고 일반화하는지 이해하는 데 둔다. 핵심은 다음 세 가지다.

1. 딥러닝은 원시 입력을 여러 층의 표현으로 변환하는 함수 근사 시스템이다.
2. 학습은 손실함수를 최소화하는 파라미터를 찾는 과정이고, 일반화는 보지 못한 데이터에서도 성능이 유지되는지를 뜻한다.
3. 보안 취약성은 모델이 학습한 표현, 데이터 분포, 출력 인터페이스, 평가 가정이 공격자에게 어떻게 노출되는지와 연결된다.

## 3. 수식 또는 알고리즘

### 3.1 경험위험 최소화

```text
Given training data D = {(x_i, y_i)}:
minimize_theta  (1/n) * sum_i L(f_theta(x_i), y_i)
```

이 식은 모델 `f_theta`가 학습 데이터에서 손실 `L`을 줄이도록 파라미터 `theta`를 찾는다는 뜻이다. 보안 평가에서는 이 목적함수가 정상 데이터 성능만 최적화할 때 공격 데이터나 프라이버시 누출에 취약할 수 있음을 함께 본다.

### 3.2 일반화 격차

```text
generalization_gap = test_loss - train_loss
```

격차가 크면 모델이 학습 데이터를 과하게 외웠을 가능성이 있다. 이는 성능 문제일 뿐 아니라 membership inference 같은 프라이버시 공격의 위험 신호가 될 수 있다.

### 3.3 대적 위험

```text
adversarial_risk = E[max_delta L(f_theta(x + delta), y)]
subject to delta in allowed_perturbation_set
```

공격자가 허용된 범위의 입력 변화 `delta`를 선택할 수 있다고 가정하면, 모델 평가는 정상 입력뿐 아니라 최악의 교란 입력에서도 수행되어야 한다.

## 4. 초보자용 설명

딥러닝은 입력을 받아 정답을 외우는 기계가 아니라, 입력을 여러 단계의 특징으로 바꿔 가며 규칙을 찾는 시스템이다. 문제는 그 규칙이 항상 사람의 직관과 같지 않다는 데 있다. 모델은 정상 데이터에서는 잘 맞지만, 사람이 보기에는 거의 같은 입력을 다르게 판단하거나, 출력 확률만으로 학습 데이터의 흔적을 드러낼 수 있다.

## 5. 보안 연구와의 연결

AI 원리는 보안 평가의 기준이 된다. 역전파와 표현학습은 adversarial example을 이해하는 출발점이고, 일반화와 overfitting은 privacy leakage를 설명하는 단서다. 생명주기 관점은 공격이 학습 이전 데이터 단계에서 생길 수도 있고, 배포 이후 질의 인터페이스에서 생길 수도 있음을 보여준다.

## 6. W01 결론

W01에서 딥러닝은 “정확도를 내는 모델”이 아니라 “데이터, 학습, 평가, 배포가 연결된 시스템”으로 정의한다. 이후 주차의 공격과 방어는 이 시스템의 어느 단계와 자산을 겨냥하는지에 따라 정리한다.
