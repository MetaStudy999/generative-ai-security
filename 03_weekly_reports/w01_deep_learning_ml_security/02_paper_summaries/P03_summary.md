### 3.1 이진 침입탐지 문제

침입탐지는 관측 이벤트 또는 트래픽 특성 $x$를 정상 또는 공격 클래스로 분류하는 문제로 표현할 수 있다.

$$
\hat{y} = f_{\theta}(x), \qquad \hat{y} \in \{0,1\}
$$

| 기호 | 의미 |
|---|---|
| $x$ | 네트워크 패킷, 플로우, 호스트 로그, 시스템 이벤트 등 관측 데이터 |
| $f_{\theta}$ | 파라미터 $\theta$를 가진 탐지 모델 |
| $\hat{y}$ | 모델이 예측한 클래스 |
| $0$ | 정상 또는 benign 클래스 |
| $1$ | 침입, 공격, malicious 클래스 |

### 보안적 의미

침입탐지는 단순 분류 문제가 아니다. 정상 트래픽과 공격 트래픽은 시간에 따라 변하고, 새로운 공격은 학습 데이터에 없을 수 있으며, 공격 데이터는 정상 데이터보다 훨씬 적은 경우가 많다. 따라서 모델의 평균 정확도보다 **공격 탐지 능력과 오탐 비용**을 함께 평가해야 한다.

---

### 3.2 Confusion Matrix 기반 탐지 지표

침입탐지 성능은 보통 $TP$, $TN$, $FP$, $FN$으로 구성된 confusion matrix에서 계산된다.

| 구분 | 예측: 공격 | 예측: 정상 |
|---|---:|---:|
| 실제: 공격 | $TP$ | $FN$ |
| 실제: 정상 | $FP$ | $TN$ |

| 기호 | 의미 |
|---|---|
| $TP$ | 공격을 공격으로 올바르게 탐지한 수 |
| $TN$ | 정상 이벤트를 정상으로 올바르게 분류한 수 |
| $FP$ | 정상 이벤트를 공격으로 잘못 탐지한 수. 오탐 |
| $FN$ | 공격 이벤트를 정상으로 잘못 분류한 수. 미탐 |

---

### 3.3 Precision, Recall, F1-score

보안 탐지에서는 accuracy만으로 성능을 판단하면 위험하다. 특히 공격 데이터가 적은 불균형 데이터에서는 precision, recall, F1-score가 중요하다.

$$
Precision = \frac{TP}{TP + FP}
$$

$$
Recall = \frac{TP}{TP + FN}
$$

$$
F1 = \frac{2 \cdot Precision \cdot Recall}{Precision + Recall}
$$

| 지표 | 의미 | 보안적 해석 |
|---|---|---|
| $Precision$ | 공격으로 탐지한 것 중 실제 공격의 비율 | 낮으면 보안관제 오탐 부담이 커진다. |
| $Recall$ | 실제 공격 중 탐지한 비율 | 낮으면 실제 침해를 놓칠 위험이 커진다. |
| $F1$ | precision과 recall의 조화평균 | 오탐과 미탐 균형을 볼 때 유용하다. |

### 보안적 의미

보안 운영에서는 $FP$와 $FN$의 비용이 다르다. $FP$가 많으면 관제 피로도가 증가하고, $FN$이 많으면 실제 침입을 놓친다. 따라서 침입탐지 모델의 평가는 단일 accuracy보다 precision, recall, F1, false alarm rate, detection rate를 함께 봐야 한다.

---

### 3.4 False Alarm Rate와 Detection Rate

침입탐지 분야에서는 false alarm rate와 detection rate가 핵심 지표로 사용된다.

$$
Detection\ Rate = \frac{TP}{TP + FN}
$$

$$
False\ Alarm\ Rate = \frac{FP}{FP + TN}
$$

| 지표 | 의미 | 운영상 의미 |
|---|---|---|
| $Detection\ Rate$ | 실제 공격을 탐지한 비율 | 낮으면 침해 사고를 놓칠 가능성이 커진다. |
| $False\ Alarm\ Rate$ | 정상 이벤트를 공격으로 잘못 판단한 비율 | 높으면 관제 인력의 신뢰와 대응 효율이 떨어진다. |

## 4. AI 원리 관점 분석

| 항목 | 분석 |
|---|---|
| 지도학습 | 라벨이 있는 정상/공격 데이터를 학습해 분류 모델을 만든다. SVM, decision tree, neural network 등이 사용될 수 있다. |
| 비지도학습 | 정상 패턴 또는 데이터 군집 구조를 학습한 뒤 이상치를 탐지한다. 새로운 공격 탐지에 유리하지만 오탐 관리가 어렵다. |
| 반지도학습 | 일부 라벨 데이터와 다수의 비라벨 데이터를 함께 사용한다. 실제 보안 데이터의 라벨 부족 문제와 연결된다. |
| 앙상블 | 여러 모델을 결합해 탐지 성능과 안정성을 높인다. 단, 해석성과 운영 비용이 증가할 수 있다. |
| 특징공학 | 패킷, 플로우, 로그에서 통계량과 행동 특징을 추출한다. 잘못된 특징 설계는 모델 성능을 제한한다. |
| 데이터 불균형 | 정상 이벤트가 공격 이벤트보다 훨씬 많아 accuracy가 과대평가될 수 있다. |
| 일반화 | 특정 벤치마크 데이터셋에서 높은 성능을 보인 모델이 실제 운영망에서 같은 성능을 보장하지 않는다. |

## 5. 보안 이슈 관점 분석

P03은 W01의 “AI 원리 70%”가 실제 사이버보안 탐지 문제로 내려오는 핵심 사례이다. 침입탐지는 ML 모델을 실제 보안 운영에 투입할 때 발생하는 데이터, 지표, 운영 리스크를 잘 보여준다.

| 보안 항목 | 침입탐지에서의 의미 |
|---|---|
| 기밀성 | 침입 탐지 실패는 내부 정보 유출로 이어질 수 있다. |
| 무결성 | 공격자가 로그나 트래픽을 조작하면 탐지 모델의 판단이 왜곡될 수 있다. |
| 가용성 | 과도한 오탐은 관제 시스템과 대응 인력을 마비시킬 수 있다. |
| 프라이버시 | 네트워크·호스트 로그에는 개인정보와 민감정보가 포함될 수 있다. |
| 책임성 | 탐지 결과, 경보, 대응 이력은 사후 감사 가능해야 한다. |
| 운영 리스크 | 데이터 drift, 새로운 공격, 암호화 트래픽, 라벨 오류가 모델 신뢰성을 낮춘다. |

## 6. 위협모형

### 6.1 보호 대상

| 보호 대상 | 설명 |
|---|---|
| 네트워크 트래픽 | 패킷, 플로우, 연결 정보, 프로토콜 메타데이터 |
| 호스트 로그 | 시스템 호출, 프로세스 실행, 파일 접근, 인증 로그 |
| 탐지 모델 | 분류기, 이상탐지기, 앙상블 모델, threshold 설정 |
| 경보 시스템 | alert, severity, correlation rule, 대응 워크플로우 |
| 운영 데이터 | 실시간 이벤트 스트림, 라벨링 결과, incident 기록 |

### 6.2 공격자 능력

| 공격자 유형 | 가능 행위 |
|---|---|
| Evasion attacker | 탐지 모델이 정상으로 판단하도록 입력 패턴을 조정한다. |
| Data poisoning attacker | 학습 데이터 또는 라벨을 오염시켜 모델 판단을 왜곡한다. |
| Low-and-slow attacker | 탐지 threshold를 넘지 않도록 공격 행위를 분산한다. |
| Drift exploiter | 운영 환경 변화와 공격 변종을 이용해 모델의 과거 학습 한계를 노린다. |
| Log tampering attacker | 탐지 또는 사후 분석에 필요한 로그를 누락·변조한다. |

### 6.3 공격 경로

```text
보안 이벤트 수집
→ 특징 추출 및 라벨링
→ 탐지 모델 학습
→ 운영 트래픽 분류
→ 공격자가 입력 패턴, 데이터 품질, 로그 무결성을 교란
→ 오탐 증가 또는 미탐 발생
```

## 7. 평가방법 및 지표

| 지표 | 의미 | W01/P03에서의 활용 |
|---|---|---|
| Accuracy | 전체 이벤트 중 맞게 분류한 비율 | 불균형 데이터에서는 단독 사용 금지 |
| Detection Rate / Recall | 실제 공격 중 탐지한 비율 | 미탐 위험 평가 |
| False Alarm Rate | 정상 이벤트 중 공격으로 잘못 탐지한 비율 | 관제 피로도와 운영 비용 평가 |

| ROC-AUC | threshold 변화에 따른 탐지 성능 | threshold 조정과 운영 정책 검토 |
| PR-AUC | 불균형 데이터에서 precision-recall 관계 평가 | 공격 데이터가 적을 때 유용 |
| Latency | 이벤트 발생 후 탐지까지 걸리는 시간 | 실시간 탐지 가능성 평가 |
| Drift Metric | 운영 데이터와 학습 데이터의 분포 차이 | 재학습 또는 모델 갱신 판단 |

## 8. 재현성 점검

이 논문은 survey 논문이므로 특정 단일 실험을 재현하는 방식보다는, 공개 IDS 데이터셋과 표준 분류기를 이용해 평가 지표 체계를 재현하는 방식이 적절하다.

| 항목 | 점검 |
|---|---|
| 데이터셋 | KDD Cup 99, NSL-KDD, UNSW-NB15, CICIDS 계열 등 공개 데이터셋 사용 가능. 단, 데이터셋 노후화와 중복 문제를 명시해야 한다. |
| 모델 | Decision Tree, Random Forest, SVM, Logistic Regression, MLP 등으로 최소 재현 가능 |
| 전처리 | 범주형 인코딩, 정규화, train/test split, class imbalance 처리 기록 필요 |
| seed/config | split seed, 모델 hyperparameter, threshold 설정 기록 필요 |
| 결과 파일 | confusion matrix, precision, recall, F1, FAR, detection rate를 CSV/JSON으로 저장 |
| 재현 가능성 판단 | 표준 데이터셋 기반 지표 재현은 가능. 실제 운영망 일반화는 제한적 |

### W01 실습 연결

W01에서는 다음 최소 실험으로 P03의 핵심 평가 관점을 재현할 수 있다.

1. 공개 IDS 또는 toy classification 데이터셋을 로딩한다.
2. 정상/공격 라벨 분포를 확인한다.
3. 기본 분류기를 학습한다.
4. accuracy, precision, recall, F1, false alarm rate를 함께 계산한다.
5. class imbalance 조건에서 accuracy만 볼 때 생기는 왜곡을 분석한다.
6. 결과를 confusion matrix와 metric table로 저장한다.

| 공격자 적응성 부족 | 많은 IDS 연구는 공격자가 탐지 모델을 회피하도록 적응하는 상황을 충분히 다루지 않는다. | P04 대적 공격 summary와 결합한다. |
| 프라이버시 고려 부족 | 보안 로그와 트래픽 데이터에는 민감정보가 포함될 수 있다. | P05 프라이버시 공격 summary와 결합한다. |

## 12. 기말논문 연결 3문장

1. 이 주차에서 기말논문에 반영할 개념: 침입탐지는 AI 보안 평가에서 accuracy만으로는 부족하며, 오탐과 미탐의 비용을 구분해야 한다는 핵심 사례이다.
2. 이 주차에서 기말논문에 반영할 표·그림·실험: confusion matrix, precision/recall/F1 수식, false alarm rate 계산표, IDS 평가 지표 비교표를 반영한다.
3. 이 주차가 RAG 문서 오염/LLM 보안 감사 프레임워크와 연결되는 지점: IDS의 오탐·미탐 개념은 RAG/LLM 보안에서도 위험 문서 탐지의 FPR/FNR, 공격 성공률, 감사 로그 품질 평가로 확장할 수 있다.
