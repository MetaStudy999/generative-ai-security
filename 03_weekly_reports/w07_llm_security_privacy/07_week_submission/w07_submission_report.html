초록

본 보고서는 LLM의 pretraining, instruction tuning, alignment, evaluation benchmark를 AI 원리 관점에서 정리하고, training data extraction, prompt injection, prompt leakage, privacy leakage, insecure code generation을 보안 평가 관점에서 분석한다. 실습에서는 실제 LLM/API를 호출하지 않고 synthetic prompt category와 toy guard score simulator를 사용했다.

키워드: LLM evaluation, alignment, prompt injection, privacy leakage, refusal quality, code security, reproducibility

AI 원리 70%와 보안 이슈 30%
