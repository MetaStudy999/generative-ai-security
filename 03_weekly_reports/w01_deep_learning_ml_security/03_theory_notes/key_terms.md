# 핵심 용어

| 용어 | 작업 정의 | 관련 논문 | W01 활용 |
|---|---|---|---|
| 딥러닝 패러다임 | 다층 신경망이 데이터의 계층적 표현을 학습하는 AI 접근 | P01 | 배경 이론 |
| 표현학습 | 원시 입력에서 과제에 유용한 특징을 모델이 직접 학습하는 과정 | P01 | adversarial vulnerability 설명 |
| 역전파 | 손실함수의 gradient를 뒤로 전파해 파라미터를 갱신하는 알고리즘 | P01 | gradient 기반 공격의 원리 배경 |
| 일반화 | 학습 데이터 밖의 새 데이터에서도 성능이 유지되는 성질 | P01, P02 | clean/robust 성능 구분 |
| 과적합 | 학습 데이터에는 잘 맞지만 새 데이터에는 취약한 상태 | P01, P05 | membership inference 위험 신호 |
| ML 생명주기 | 데이터 관리, 학습, 검증, 배포, 모니터링을 포함한 반복 과정 | P02 | 위협모형의 단계 구분 |
| 보증 증거 | 시스템이 요구사항을 만족한다는 주장을 뒷받침하는 기록과 검증 결과 | P02 | Docker, seed, config, DOI 검증 |
| 침입탐지 | 정상 행위와 공격 행위를 식별하거나 이상행위를 탐지하는 보안 기능 | P03 | 보안 데이터 평가 사례 |
| 오탐 | 정상 행위를 공격으로 잘못 탐지하는 오류 | P03 | 운영 비용 평가 |
| 미탐 | 공격 행위를 정상으로 잘못 판단하는 오류 | P03 | 보안 실패 비용 평가 |
| 대적 예제 | 모델 예측을 바꾸도록 조작된 입력 | P04 | 추론 단계 무결성 위협 |
| 공격 성공률 | 공격 조건에서 목표 오분류나 회피가 성공한 비율 | P04 | robust 평가 지표 |
| Robust accuracy | 공격 또는 교란 조건에서도 유지되는 정확도 | P04 | clean accuracy와 비교 |
| Membership inference | 특정 샘플이 학습 데이터에 포함됐는지 추론하는 공격 | P05 | privacy leakage 평가 |
| Model inversion | 모델 출력으로 입력 특성이나 민감 정보를 복원하려는 공격 | P05 | 기밀성 위협 |
| Differential privacy | 단일 데이터 포인트의 포함 여부가 출력에 미치는 영향을 제한하는 보호 원리 | P05 | 방어 후보 |
| 재현성 | 같은 코드, 데이터, 설정으로 결과를 다시 만들 수 있는 성질 | P02-P05 | 최종 제출 체크리스트 |
