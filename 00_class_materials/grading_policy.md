# 수업운영 및 평가 기준

## 1. 수업 운영

| 항목 | 기준 |
|---|---|
| 수업 형태 | 온라인 실시간 세미나 |
| 운영 시간 | 매주 토요일, 총 100분 |
| 수업 구성 | 전반 50분 교수 강의, 후반 50분 학생 발표·토론·리뷰 |
| 학습 비율 | AI 원리·이론 약 70%, 보안 이슈 약 30% |
| 주교재 | 매주 SCI/SCIE 논문 5편 |
| 기본 트랙 | Research Track, AI Tool Track |

Research Track은 연구문제, 위협모형, 평가방법, 재현성, 한계/오픈문제를 중심으로 운영한다. AI Tool Track은 사용 도구, 사용 목적, 주요 프롬프트, 반영 위치, 사람 검토와 검증 방식을 기록한다.

## 2. 발표 기준

평가는 발표 기반 peer scoring 중심으로 운영한다.

| 항목 | 기준 |
|---|---|
| 점수 산정 | 실제 수업에 참여한 학생이 발표 종료 후 점수 제출 |
| 점수 취합 | 과대가 점수를 종합·정규화·요약통계 처리 후 익명 전달 |
| 익명성 | 개별 채점자의 식별정보는 전달하지 않음 |
| 성적 부여 | 발표 점수 분포와 학습활동 수준을 반영해 A+ ~ F 부여 |
| 불이익 | 발표 미수행 시 하위등급 처리 또는 추가구제기회 제한 가능 |
| 가산점 | 과대 등 수업운영 지원자는 일부 가산점 가능 |

발표자는 논문 5편의 연구문제, 위협모형, 평가방법, 한계, 기말논문 연결 가능성을 설명해야 한다. 수식 또는 알고리즘이 핵심인 분야는 기호, 직관적 의미, 보안 해석, 평가 지표 연결을 함께 설명한다.

## 3. 보고서 기준

주차별 보고서는 다음 항목을 포함한다.

| 항목 | 기준 |
|---|---|
| 논문 요약 | 주차별 논문 5편의 서지, 연구문제, 방법론, 결과, 한계 정리 |
| 논문 비교표 | 연구문제, 핵심 방법, 데이터/실험, 보안 위협, 평가 지표, 한계, 내 논문 활용 비교 |
| 이론정리 | AI 원리 70%, 보안 이슈 30% 구조 유지 |
| 수식·알고리즘 | 필요한 경우에만 원문 또는 검증 가능한 표준 수식 기준으로 설명 |
| 수식 작성 도구 | Markdown + LaTeX math를 기본으로 하며, 필요 시 Pandoc, KaTeX, MathJax, LaTeX/TeX Live, sympy 활용 |
| 실험보고서 | 실행 전은 `실행 전`, 실행 후는 `outputs/run_log.md`, `metrics_summary.csv`, `results.json`과 일치 |
| AI 활용기록 | 사용 도구, 목적, 주요 프롬프트, 반영 위치, 검증 방식 기록 |
| 기말논문 연결 | 주차별 보고서가 기말논문의 어느 장에 연결되는지 기록 |

기말고사 보고서는 국내 학회지 양식 기반 모의투고 논문으로 작성한다. 최종 원고에는 국내 논문 3편 이상, 해외 논문 5편 이상, 주차별 보고서 2개 이상, 표 1개 이상, 그림 1개 이상, AI 활용 고지서, 참고문헌 검증표, 학회지 양식 출처표를 포함한다.

## 4. 감점 기준

| 감점 사유 | 감점 범위 |
|---|---:|
| 학회지 양식 미사용 | -10점 |
| 분량 미달 또는 과도한 분량 초과 | -5~-10점 |
| 표 또는 그림 누락 | 각 -5점 |
| 국내 3편·해외 5편 참고문헌 기준 미달 | 각 -5점 |
| 기존 주차별 보고서 2개 이상 미반영 | -10점 |
| AI 활용 고지서 미제출 | -10점 |
| 참고문헌 검증표 미제출 | -10점 |
| 본문 인용과 참고문헌 목록 표기 차이 | -5~-15점 |
| 존재하지 않는 논문 또는 DOI 인용 | -20점 또는 0점 가능 |
| 확인하지 못한 수식·정량값을 확정값처럼 작성 | -5~-15점 |
| 표절, 대리작성, 실험결과 조작 | 0점 가능 |

| □ | 필요한 핵심 수식 또는 알고리즘은 기호, 직관, 보안 해석, 평가 지표 연결을 설명했다. |
