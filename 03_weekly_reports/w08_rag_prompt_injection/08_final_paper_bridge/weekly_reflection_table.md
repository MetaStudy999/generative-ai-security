# W08 주차별 성찰표

| 항목 | 내용 |
|---|---|
| 이번 주 핵심 배움 | RAG 보안은 검색 성능 문제가 아니라 외부 context 신뢰성, prompt boundary, tool permission이 결합된 시스템 보안 문제다. |
| 가장 중요한 논문 | P04 prompt injection survey와 P05 의료 LLM 취약성 연구 |
| 기말논문 연결 | RAG 간접 인젝션 대응을 위한 출처 검증·approval gate 평가 프레임워크 |
| 실험에서 얻은 통찰 | Source filter는 ASR을 0.575000에서 0.050000으로 낮췄고, human approval은 tool misuse를 0.000000으로 낮췄다. |
| 남은 의문 | 실제 GraphRAG에서 node/edge provenance를 어떻게 표준화할 것인가 |
| 다음 주차와 연결 | W09/W10 이후 agent, federated, privacy 평가에서도 approval/logging/reproducibility 관점을 이어갈 수 있다. |
