AI 원리와 보안 이슈

구분발표 내용
AI 원리표현학습, 역전파, 일반화, 과적합을 보안 취약성의 기술적 배경으로 설명한다.
Confidentialitymembership inference, model inversion 등으로 학습 데이터와 민감 정보가 노출될 수 있다.
Integrityadversarial example, poisoning, label noise는 모델 판단을 조작한다.
AvailabilityIDS 미탐, 오탐 폭증, drift 미탐은 운영 가능성을 낮춘다.
Accountabilityseed, config, 로그가 없으면 보안 주장을 재검토하기 어렵다.

결론 메시지

  1. 딥러닝 원리는 보안 취약성의 기술적 배경이다.
  2. ML 보안 평가는 모델 하나가 아니라 생명주기 전체를 대상으로 해야 한다.
  3. clean 성능, robust 성능, privacy leakage, reproducibility evidence를 분리해야 한다.
  4. 실험 수치는 실행 로그와 CSV/JSON 근거가 있을 때만 주장한다.
