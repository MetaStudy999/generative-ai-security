# 개인정보 및 민감정보 점검표

| 확인 | 점검 항목 |
|---|---|
| □ | 데이터셋에 실제 이름, 고객명, 계정, 이메일, 전화번호가 포함되지 않는다. |
| □ | synthetic 이메일은 `example.invalid` 도메인을 사용한다. |
| □ | synthetic 토큰과 API 키는 실제 인증정보가 아닌 placeholder임을 명시한다. |
| □ | 실험 코드는 외부 API나 실제 기업 시스템을 호출하지 않는다. |
| □ | 원고에는 실제 운영 로그나 민감 문서 내용이 포함되지 않는다. |
| □ | 최종 제출 전 개인정보 포함 여부를 검색으로 재확인한다. |
