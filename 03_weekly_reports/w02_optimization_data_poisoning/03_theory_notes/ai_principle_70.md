# AI 원리 70% 정리

W02의 AI 원리는 대규모 최적화, 효율적 학습, 압축, 경량화, 일반화이다. 보안 이슈는 중요하지만, 먼저 모델이 어떻게 학습되고 어떤 비용 구조를 갖는지 이해해야 데이터 오염 공격의 영향을 설명할 수 있다.

## 1. 핵심 이론

대규모 머신러닝은 많은 데이터와 고차원 파라미터에서 손실함수를 최소화하는 문제로 볼 수 있다. 전체 데이터의 gradient를 매번 정확히 계산하면 안정적일 수 있지만 비용이 크다. 그래서 실제 학습에서는 stochastic gradient descent와 mini-batch update가 널리 사용된다. 이 방식은 빠르고 확장성이 높지만, 샘플링 노이즈와 데이터 품질에 민감하다.

Poisoning은 바로 이 지점을 공격한다. 학습 데이터 일부가 조작되면 empirical risk와 gradient 추정이 달라지고, 모델의 decision boundary가 공격자에게 유리한 방향으로 이동할 수 있다. 따라서 W02의 핵심 연결은 “최적화는 데이터에 의해 움직이고, 데이터 오염은 최적화 경로와 최종 모델을 바꾼다”는 문장으로 요약된다.

효율적 딥러닝은 모델을 작고 빠르게 만들기 위한 압축, pruning, quantization, knowledge distillation, efficient architecture, efficient training을 포함한다. 효율화는 배포 가능성을 높이지만, 보안 평가 관점에서는 방어 비용, 탐지 지연시간, 모델 압축 후 backdoor 잔존 여부를 함께 점검해야 한다.

## 3. 수식 또는 알고리즘

Empirical risk minimization은 다음처럼 쓸 수 있다.

```text
min_w  1/n * sum_i L(f_w(x_i), y_i) + lambda * R(w)
```

여기서 `x_i`, `y_i`는 학습 데이터와 라벨이고, `w`는 모델 파라미터다. Poisoning은 `x_i` 또는 `y_i` 일부를 조작해 위 목적함수의 모양 자체를 바꾼다.

SGD의 기본 업데이트는 다음과 같다.

```text
w_{t+1} = w_t - eta_t * gradient L(f_w(x_b), y_b)
```

`x_b`, `y_b`는 mini-batch다. 오염 샘플이 mini-batch에 포함되면 gradient 방향이 달라질 수 있고, 반복 학습 과정에서 그 영향이 누적된다.

## 4. 초보자용 설명

모델 학습은 학생이 문제집을 보고 풀이 기준을 익히는 과정과 비슷하다. 문제집이 크면 모든 문제를 매번 다 풀 수 없어서 일부 문제를 보고 조금씩 실력을 조정한다. 그런데 문제집 일부의 정답이 잘못되어 있거나 특정 표시가 붙은 문제만 엉뚱한 답으로 학습하도록 되어 있다면, 학생은 평소에는 잘 풀어도 특정 조건에서 틀리는 습관을 배울 수 있다. 이것이 label-flipping과 backdoor의 직관이다.

## 5. 보안 연구와의 연결

| AI 원리 | 보안 연결 | 평가 질문 |
|---|---|---|
| SGD/mini-batch 학습 | 오염 샘플이 gradient를 왜곡할 수 있음 | 오염률이 올라가면 accuracy가 얼마나 떨어지는가 |
| 일반화 | clean test 성능과 공격 조건 성능이 다를 수 있음 | 정상 테스트셋과 trigger 테스트셋을 분리했는가 |
| 모델 압축 | 방어 비용은 줄일 수 있지만 backdoor 잔존 가능성 존재 | 압축 전후 ASR과 detection rate를 비교했는가 |
| 효율적 학습 | 빠른 재학습과 반복 평가에 유리 | 방어 재학습 비용과 로그가 기록되었는가 |
| 재현성 | seed/config/log 없이는 보안 주장을 검토하기 어려움 | Docker, config, 데이터 버전, 실행 로그가 남았는가 |
