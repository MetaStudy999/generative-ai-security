본 보고서는 자기지도학습과 파운데이션 모델의 사전학습 표현공간을 보안 자산으로 보고, poisoning/backdoor 위협을 평가하기 위한 다중지표 구조를 정리한다. synthetic 2차원 표현공간 클러스터 기반 안전 toy 실험으로 clean accuracy, poisoned clean accuracy, ASR, mean shift, detection rate, clean FPR을 분리 기록했다.

1. 한 문장 요약

라벨 없는 pretraining 환경에서도 데이터 수집, augmentation, pair 구성, representation space가 공격면이 될 수 있으므로 clean accuracy와 ASR을 분리해 보고해야 한다.

2. 학습 배경과 주차 목표

W05는 W01-W04의 생명주기 보안, 데이터 오염, 표현공간 교란, Transformer/NLP 보안 논의를 SSL/foundation pretraining 단계로 확장한다. 학습목표는 contrastive/masked/predictive SSL 이해, 비디오 SSL 전이평가 이해, poisoning/backdoor representation 평가축 설계다.

3. AI 원리 70% 정리

자기지도학습은 라벨 없이 contrastive, generative, predictive objective를 통해 표현을 학습한다[1]. 추천 SSL과 비디오 SSL은 도메인별 데이터 구조와 temporal/cross-modal signal을 활용한다[2][3].

표 1. W05 핵심 개념과 보안 연결
개념AI 원리보안 연결
Contrastive learningpositive/negative pair로 표현공간 구조를 만든다pair 오염, augmentation poisoning
Representation learningdownstream task에 전이 가능한 embedding을 만든다representation shift

4. 보안 이슈 30% 정리

Poisoning은 학습 데이터 조작으로 최적화와 판단을 왜곡하고[4], backdoor는 clean accuracy가 유지되어도 trigger 조건에서 ASR이 높아질 수 있다[5].

그림 1. 자기지도학습 기반 표현공간 보안 평가 흐름

6. 논문 5편 비교표

P01-P03은 SSL/표현학습 원리와 응용 문헌이고, P04-P05는 poisoning/backdoor 보안 문헌이다. W05 toy 실험은 실제 SSL/foundation model 실험이 아니라 표현공간 평가축 설명용이다.

14. 발표용 요약

표현학습 기반 모델의 보안 평가는 clean 성능 하나로 끝나지 않으며, trigger 조건의 ASR, representation shift, defense check, 재현성 근거를 분리해야 한다.
