# 기말과제 요약

## 1. 과제명

AI 보안 국내 학술지 모의투고 논문 작성

## 2. 핵심 목표

한 학기 동안 읽고 분석한 논문과 주차별 보고서를 바탕으로 작은 AI 보안 연구문제를 정의하고, 국내 학회지 형식에 맞춘 모의투고 논문 원고를 작성한다.

## 3. 주요 제출물

- 최종 논문 원고
- PDF 변환본
- 학회지 양식 출처표
- AI 활용 고지서
- 참고문헌 검증표
- 주차별 보고서 반영표

## 4. 작성 기준

| 항목 | 기준 |
|---|---|
| 논문 형식 | 국내 학회지 또는 논문지 공식 HWP/DOCX 양식 사용 |
| 분량 | A4 기준 6쪽 이상 12쪽 이하, 단 선택 학회지 기준이 더 엄격하면 그 기준 우선 |
| 연구범위 | AI 보안, 생성형 AI 보안, 드론·로봇 보안, 피지컬 AI 보안, CPS 보안, RAG 보안, AI 에이전트 보안, 프라이버시 보존형 AI 등 |
| 참고문헌 | 국내 논문 3편 이상, 해외 논문 5편 이상 실제 검증 |
| 주차별 보고서 반영 | 최소 2개 이상을 기말논문 본문에 반영하고 반영표 작성 |
| 표·그림 | 표 1개 이상, 그림 1개 이상 포함 |
| AI 활용 고지 | 사용 도구, 목적, 주요 프롬프트, 반영 위치, 검증 방식 공개 |
| 연구윤리 | 허위 인용, 존재하지 않는 DOI, 표절, 실행하지 않은 실험 결과 작성 금지 |

## 5. 수식·알고리즘 설명 기준

기말논문은 국내 학술지 형식의 논문이므로, 각 분야의 핵심 수식이나 알고리즘이 연구문제와 직접 연결되는 경우에는 본문 또는 표로 설명하는 것이 좋다. 단순 장식용 수식은 넣지 않는다.

반영이 필요한 경우:

- 논문 기여가 손실함수, 최적화, privacy budget, aggregation, reward, perturbation, 검증 조건처럼 수식으로 정의되는 경우
- 공격 성공률, robust accuracy, privacy leakage, utility, cost 등 평가 지표를 해석해야 하는 경우
- 독자가 수식을 모르면 보안 위협 또는 방어 원리를 이해하기 어려운 경우

작성 방식:

1. 원문 또는 검증 가능한 표준 수식을 제시한다.
2. 각 기호의 의미와 단위를 설명한다.
3. 수식이 연구문제, 위협모형, 평가 지표 중 어디에 연결되는지 밝힌다.
4. 초보자용 직관 설명을 덧붙인다.
5. 원문 확인이 부족한 수식은 `확인 필요`로 표시하고 임의로 만들지 않는다.

## 6. 권장 도구와 라이브러리

| 용도 | 권장 도구 |
|---|---|
| 기본 수식 작성 | Markdown + LaTeX math |
| HTML 렌더링 | KaTeX 또는 MathJax |
| DOCX/PDF 변환 | Pandoc |
| 최종 PDF 품질 관리 | LaTeX / TeX Live |
| 수식 검산 | sympy |
| 수치 계산 | numpy, scipy |
| 그래프와 그림 수식 | matplotlib |
| 결과표 정리 | pandas |
| 대화형 검토 | jupyterlab |

상세 기준은 `math_formula_toolchain.md`를 참고한다. 선택 라이브러리는 실제 실험이나 문서 변환에 필요할 때만 프로젝트 의존성에 추가하고, 추가 시 lock 파일과 실행 로그를 함께 갱신한다.

현재 단계에서는 추가 설치가 필수는 아니다. 기본 작성은 Markdown + LaTeX math로 진행하고, 수식 검산이 필요할 때 `sympy`를 먼저 선택 추가한다. `scipy`, `jupyterlab`, Pandoc, TeX Live는 실제 계산·노트북 검토·문서 변환 요구가 생길 때 추가한다.
