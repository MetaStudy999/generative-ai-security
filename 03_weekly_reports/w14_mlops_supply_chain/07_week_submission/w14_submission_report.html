1. 한 문장 요약

운영형 AI 시스템 보안은 모델 accuracy만이 아니라 dataset hash, config hash, model hash, drift score, audit coverage, artifact inventory, re-run consistency가 함께 남는 evidence set 문제다.

초록: 본 보고서는 MLOps/DevOps, data/model pipeline, monitoring, drift detection을 AI 원리로 정리하고, ML supply chain, artifact tampering, logging/update attack surface를 보안 이슈로 분석한다. Synthetic MLOps toy pipeline 기반 안전 실험에서는 baseline accuracy 0.925000, F1 0.923077, drift score 0.307626, audit coverage 1.000000, inventory coverage 1.000000을 기록했다. 이 수치는 실제 기업 공급망 보안 보증이 아니라 evidence set 보고 형식 검증용 결과다.

2. 학습 배경과 주차 목표

W14는 모델 단위 AI 보안 평가를 운영형 AI 시스템과 공급망 보안으로 확장한다. 목표는 MLOps lifecycle, ML supply chain attack surface, drift monitoring, AI BOM/ML artifact inventory, audit evidence set을 함께 설명하는 것이다.

3. AI 원리 70% 정리

MLOps는 ML, software engineering, data engineering, operations를 연결한다. Production ML deployment, AIOps, edge deployment, deep learning for software engineering 문헌을 통해 data/model/config/log/artifact traceability의 필요성을 정리했다.
