# 주제 후보

| 번호 | 주제 후보 | 대상 | 핵심 위협 | 접근법 | 장점 | 주의점 |
|---:|---|---|---|---|---|---|
| 1 | ML 생명주기 기반 AI 보안 평가 프레임워크 | AI/ML 시스템 전체 | 데이터 오염, 대적 입력, 프라이버시 누출, 재현성 실패 | 문헌분석, 평가 체크리스트, 주차별 매핑 | 모든 주차를 묶기 좋음 | 범위가 넓어질 수 있어 평가 항목을 제한해야 함 |
| 2 | Clean 성능과 Robust 성능의 분리 평가 연구 | 분류/탐지 모델 | adversarial example, 탐지 회피 | toy 실험 + 문헌 지표 비교 | 실험 설계가 명확함 | 공격 세부 절차는 안전 범위 안에서 제한 필요 |
| 3 | ML 프라이버시 공격의 위협모형과 방어 비교 | 학습 데이터와 모델 출력 | membership inference, model inversion | threat model, defense matrix | 기밀성 축을 깊게 다룰 수 있음 | 실제 개인정보 없이 synthetic/public data로 제한 |
| 4 | 보안 논문 재현성 체크리스트 연구 | 주차별 논문 패킷과 실습 산출물 | DOI 오류, 설정 누락, 실행 로그 부재 | 문헌 검증표 + Docker/config 점검 | 제출물 품질과 직접 연결 | 기술적 novelty를 보완해야 함 |

## 우선 후보

우선 후보는 1번이다. W01의 다섯 논문이 AI 원리, 생명주기, 탐지, 강건성, 프라이버시를 모두 포함하므로, 기말 논문 전체를 묶는 상위 프레임으로 가장 적합하다.
