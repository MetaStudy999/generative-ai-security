1. 한 문장 요약

W04는 Transformer 효율화, NLP 대적공격, prompt privacy를 연결하여 clean score, attack success rate, privacy leakage, utility score, reproducibility evidence를 분리 보고하는 평가 구조를 제시한다.

2. 학습 배경과 주차 목표

W04는 W03의 비전 Transformer와 robust evaluation을 NLP Transformer와 프롬프트 기반 보안 문제로 확장하는 주차이다. 이후 W07 LLM 보안, W08 RAG 프롬프트 인젝션, W11 차등프라이버시, W14 MLOps 로그 거버넌스와 연결된다.

3. AI 원리 70% 정리

Efficient Transformer 연구는 self-attention의 계산 복잡도와 긴 시퀀스 처리 비용을 줄이는 구조적 접근을 분류한다[1]. Faster/lighter Transformer 연구는 speed, memory, latency, 경량화 전략을 정리한다[2]. Transformer survey는 구조와 응용 taxonomy를 제공한다[3].

표 1. W04 핵심 개념과 보안 연결

4. 보안 이슈 30% 정리

NLP 대적공격 연구는 단어 치환, 문장 재구성, transfer attack을 다룬다[4]. Prompt privacy 연구는 프롬프트 입력과 ICL 예시의 민감정보 노출 위험을 다룬다[5].

8. 실습 보고서

본 실습은 실제 Transformer 또는 LLM 공격 재현이 아니라 W04의 핵심인 프롬프트 프라이버시 평가축을 안전하게 설명하기 위한 최소 toy protocol이다.

그림 1. 프롬프트 기반 NLP 보안 평가 흐름

User Prompt / ICL Examples
        ↓
Transformer / NLP System
        ↓
Clean Evaluation ──> Clean Score
        ↓
Word Substitution / Paraphrase
        ↓
Adversarial Evaluation ──> Attack Success Rate
        ↓
Masking / Privacy-Preserving Prompt Wrapper
        ↓
Privacy Evaluation ──> Privacy Leakage, Utility Score
        ↓
Reproducibility Evidence ──> seed, config, Docker, outputs, run_log

표 4. W04 실습 설계

항목내용
