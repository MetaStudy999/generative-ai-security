# 위협모형

| 항목 | 내용 |
|---|---|
| 대상 시스템 | LLM/RAG 기반 질의응답 및 평가 파이프라인 |
| 보호 자산 | 프롬프트, 검색 문서, 평가셋, 모델 출력, 사용자 정보, 실험 로그, 참고문헌 목록 |
| 공격자 | 외부 사용자, 악의적 문서 제공자, 데이터 제공자, 평가셋 오염 행위자, 내부 실수 |
| 공격자의 지식 | White-box / Gray-box / Black-box |
| 공격자의 능력 | 악성 지시문 삽입, 오염 문서 제공, 반복 질의, 평가셋 누수 유도, 민감정보 유출 유도 |
| 공격 경로 | 입력 프롬프트, 검색 문서, RAG context, 평가 benchmark, 로그와 제출물 |
| 공격 성공 조건 | 정책 우회, 근거 없는 답변, 평가 점수 왜곡, 민감정보 노출, 재현 불가능한 결과 |
| 방어자 가정 | 문서 출처 검증, human approval gate, config/seed/log 보존, 참고문헌 검증 가능 |
| 제외 범위 | 실제 시스템 침해, 실제 개인정보 추론, 무단 API 대량 질의 |

## 위협모형 설명

본 연구의 위협모형은 실제 공격 절차를 재현하기 위한 것이 아니라, LLM/RAG 기반 AI 시스템에서 보호해야 할 자산과 평가해야 할 실패 조건을 정의하기 위한 분석 틀이다. 공격자는 프롬프트, 검색 문서, 평가셋, 로그 등 생명주기 각 단계의 입력을 오염시킬 수 있다고 가정하지만, 실제 서비스 침해나 실제 개인정보 추론은 연구 범위에서 제외한다.
