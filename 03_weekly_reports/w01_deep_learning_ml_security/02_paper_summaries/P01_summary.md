### 3.1 경험위험최소화

딥러닝 모델의 기본 학습 목적은 학습 데이터에서 평균 손실을 최소화하는 것이다.

$$
\min_{\theta} \hat{R}(\theta)
= \frac{1}{n}\sum_{i=1}^{n} \ell(f_{\theta}(x_i), y_i)
$$

| 기호 | 의미 |
|---|---|
| $x_i$ | $i$번째 입력 데이터 |
| $y_i$ | $i$번째 정답 레이블 |
| $f_{\theta}$ | 파라미터 $\theta$를 가진 딥러닝 모델 |
| $\ell(\cdot)$ | 예측값과 정답 사이의 손실함수 |
| $\hat{R}(\theta)$ | 경험위험, 즉 학습 데이터에서의 평균 손실 |
| $n$ | 학습 데이터 수 |

### 보안적 의미

경험위험최소화는 정상 데이터 분포에서 평균 손실을 줄이는 데 초점을 둔다. 그러나 AI 보안에서는 정상 평균 성능만으로 충분하지 않다. 공격자는 전체 분포의 평균이 아니라 모델이 취약한 일부 입력 영역, 특정 트리거, 특정 데이터 포인트, 특정 API 응답 패턴을 찾는다.

| 구분 | 일반 ML 평가 | AI 보안 평가 |
|---|---|---|
| 목표 | 평균 손실 최소화 | 정상 성능과 공격 조건 성능 동시 평가 |
| 주요 지표 | Accuracy, Loss | Robust Accuracy, ASR, FPR, FNR |
| 관심 입력 | 정상 테스트 데이터 | 교란 입력, 오염 데이터, 분포 밖 입력 |
| 실패 조건 | 오분류 | 공격 성공, 민감정보 노출, 탐지 우회 |

---

### 3.2 역전파와 경사하강법

딥러닝은 손실함수의 기울기를 이용해 파라미터를 반복적으로 갱신한다.

$$
\theta_{t+1} = \theta_t - \eta \nabla_{\theta} \hat{R}(\theta_t)
$$

| 기호 | 의미 |
|---|---|
| $\theta_t$ | $t$번째 학습 단계의 모델 파라미터 |
| $\eta$ | 학습률 |
| $\nabla_{\theta}\hat{R}(\theta_t)$ | 손실함수의 파라미터 기울기 |
| $\theta_{t+1}$ | 갱신된 파라미터 |

### 보안적 의미

역전파는 딥러닝 학습의 핵심이지만, 동시에 gradient 기반 공격의 기술적 기반이 된다. 공격자는 손실을 증가시키는 방향으로 입력을 조작하거나, 학습 데이터를 오염시켜 파라미터 업데이트 방향을 왜곡할 수 있다.

대표적인 입력 교란 공격은 다음과 같이 단순화할 수 있다.

$$
x^{adv} = x + \epsilon \cdot \operatorname{sign}\left(\nabla_x \ell(f_{\theta}(x), y)\right)
$$

| 기호 | 의미 |
|---|---|
| $x$ | 정상 입력 |
| $x^{adv}$ | 공격자가 만든 대적 입력 |
| $\epsilon$ | 허용된 교란 크기 |
| $\nabla_x\ell$ | 입력에 대한 손실함수의 기울기 |
| $\operatorname{sign}(\cdot)$ | 기울기의 부호 함수 |

이 식은 딥러닝의 강점인 미분가능성이 동시에 공격면이 될 수 있음을 보여준다. 즉, 모델을 잘 학습시키는 gradient 정보는 모델을 공격하는 데에도 사용될 수 있다.

### 3.3 표현학습

딥러닝은 입력 $x$를 여러 층의 비선형 변환을 통해 점점 추상적인 표현으로 바꾼다.

$$
h^{(l)} = \sigma\left(W^{(l)}h^{(l-1)} + b^{(l)}\right)
$$

| 기호 | 의미 |
|---|---|
| $h^{(l)}$ | $l$번째 층의 은닉 표현 |
| $W^{(l)}$ | $l$번째 층의 가중치 |
| $b^{(l)}$ | $l$번째 층의 편향 |
| $\sigma(\cdot)$ | 비선형 활성화 함수 |
| $h^{(0)}$ | 원시 입력 $x$ |

### 보안적 의미

표현학습은 원시 입력을 추상적 특징으로 바꾸기 때문에 높은 성능을 가능하게 한다. 그러나 내부 표현이 특정 교란, 트리거, 분포 이동에 민감하면 모델은 정상적으로 보이는 입력에도 잘못된 예측을 할 수 있다. 따라서 AI 보안에서는 최종 출력뿐 아니라 중간 표현의 안정성, 결정경계의 여유, confidence의 신뢰성도 함께 평가해야 한다.

## 4. AI 원리 관점 분석

| 항목 | 분석 |
|---|---|
| 모델 구조 | 다층 신경망은 여러 은닉층을 통해 원시 입력을 점진적으로 추상화한다. |
| 학습 방식 | 손실함수를 정의하고 역전파를 통해 각 층의 파라미터를 갱신한다. |
| 표현학습 | 사람이 직접 특징을 설계하지 않고, 모델이 데이터로부터 유용한 표현을 학습한다. |
| CNN | 이미지·비디오·음성처럼 지역성과 계층성이 있는 데이터에 적합하다. |

| 일반화 | 대규모 데이터, 적절한 구조, 정규화, 최적화가 결합될 때 좋은 성능을 보인다. |
| 평가 | 단순 학습 정확도보다 테스트 성능, 일반화 오차, 분포 이동 성능이 중요하다. |

| 백도어 | 특정 트리거가 내부 표현과 출력에 조건부 연관을 형성하게 만든다. |

## 6. 위협모형

### 6.1 보호 대상

| 보호 대상 | 설명 |
|---|---|
| 모델 파라미터 | 학습된 가중치와 구조 |
| 학습 데이터 | 개인정보, 보안 로그, 이미지, 음성, 텍스트 |
| 추론 결과 | 분류 결과, confidence score, 생성 결과 |
| 내부 표현 | 은닉층 표현, feature embedding |
| 운영 파이프라인 | 데이터 입력, 전처리, 모델 배포, 모니터링 |

### 6.2 공격자 능력

| 공격자 수준 | 가능 행위 |
|---|---|
| Black-box | 모델 입력과 출력만 관찰한다. |
| Gray-box | 일부 confidence, API 응답, 데이터 분포를 관찰한다. |
| White-box | 모델 구조, 파라미터, gradient에 접근한다. |
| Training-time attacker | 학습 데이터 또는 라벨을 조작한다. |
| Inference-time attacker | 추론 입력을 조작하거나 반복 질의를 수행한다. |

### 6.3 공격 경로

```text
정상 데이터 학습
→ 다층 표현 형성
→ 입력-출력 결정경계 생성
→ 공격자가 입력/데이터/출력 정보를 이용
→ 대적 예제, 데이터 오염, 프라이버시 공격, 모델 추출 발생
```

## 7. 평가방법 및 지표

| 지표 | 의미 | W01/P01에서의 활용 |

| 실험 | clean accuracy, adversarial perturbation, generalization gap 측정 가능 |

## 12. 기말논문 연결 3문장

1. 이 주차에서 기말논문에 반영할 개념: 딥러닝은 원시 입력을 다층 표현으로 변환하는 함수 근사 시스템이며, 이 표현학습 구조가 AI 보안 취약성 분석의 출발점이 된다.
2. 이 주차에서 기말논문에 반영할 표·그림·실험: 딥러닝 학습 과정도, 경험위험최소화 수식, 역전파 기반 학습 흐름도, clean accuracy와 robust accuracy 비교표를 반영한다.
3. 이 주차가 RAG 문서 오염/LLM 보안 감사 프레임워크와 연결되는 지점: RAG와 LLM 보안도 결국 입력 컨텍스트가 내부 표현과 출력에 영향을 주는 구조이므로, P01의 표현학습·손실함수·일반화 개념을 W07/W08의 LLM 보안 평가 지표로 확장한다.
