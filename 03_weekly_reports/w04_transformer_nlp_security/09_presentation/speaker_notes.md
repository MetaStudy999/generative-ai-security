# W04 발표자 노트

## 1. 표지

오늘 발표는 Transformer 변형과 NLP 대적공격, 그리고 프롬프트 프라이버시를 연결합니다. 핵심 메시지는 clean score 하나로는 프롬프트 기반 NLP 보안을 설명할 수 없다는 점입니다.

## 2. 왜 중요한가

Transformer는 LLM의 기본 구조이고, 긴 프롬프트와 ICL 예시는 모델 활용성을 높입니다. 하지만 입력에 민감정보가 들어가거나 공격자가 단어를 살짝 바꾸면 탐지와 보호가 어려워질 수 있습니다.

## 3. Attention 원리

Self-attention은 토큰 사이 관계를 계산합니다. Q, K, V를 간단히 “무엇을 찾는가, 무엇과 비교하는가, 어떤 정보를 전달하는가”로 설명하면 됩니다. 이 구조가 긴 입력에서 강력하지만 비용이 큽니다.

## 4. Efficient Transformer

Sparse attention, low-rank approximation, kernelized attention은 모두 attention 비용을 줄이려는 방법입니다. 보안 관점에서는 긴 프롬프트를 평가하고 마스킹하는 비용도 함께 고려해야 합니다.

## 5. 보안 이슈

단어 치환 공격은 키워드 기반 탐지를 우회할 수 있습니다. 프롬프트 프라이버시는 사용자가 넣은 민감정보가 응답이나 로그에 남는 문제입니다. ICL 예시도 데이터처럼 관리해야 합니다.

## 6. 논문 5편의 역할

P01-P03은 Transformer 원리와 효율화 분류를 맡고, P04-P05는 NLP 대적공격 방어와 프롬프트 프라이버시를 맡습니다. 원리 문헌과 보안 문헌을 묶어야 연구문제가 선명해집니다.

## 7. 위협모형

보호 자산은 입력 문장, 프롬프트, synthetic 민감값, 응답, 로그입니다. 공격자는 단어 치환이나 민감정보 삽입으로 탐지기를 우회할 수 있고, 방어자는 마스킹과 정책 지시, 로그 보존으로 대응합니다.

## 8. 평가 프로토콜

평가 지표는 clean score, attack success rate, privacy leakage, utility score로 나눕니다. 이 네 가지를 분리해야 “정상 성능은 좋은데 공격에는 취약하다” 같은 상황을 설명할 수 있습니다.

## 9. Toy 실험

실험은 실제 개인정보나 API 없이 synthetic privacy-risk prompts로 구성했습니다. keyword privacy-risk detector를 기준으로 단어 치환 공격과 마스킹 방어를 비교했습니다.

## 10. 결과

Clean baseline은 1.000000입니다. 단어 치환 후 score는 0.625000으로 내려갔고 ASR은 0.750000입니다. 마스킹과 privacy-preserving prompt 조건의 privacy leakage는 0.000000입니다.

## 11. 한계

이 결과는 toy 실험입니다. 실제 Transformer나 LLM의 보안 성능으로 일반화하면 안 됩니다. 발표에서 중요한 것은 수치 자체보다 지표 분리와 로그 근거를 갖춘 평가 구조입니다.

## 12. 기말논문 연결

기말논문에서는 프롬프트 기반 AI 시스템의 민감정보 보호 평가체계를 제안하는 방향으로 연결할 수 있습니다. 관련연구, 위협모형, 평가방법, 실험 장에 모두 반영 가능합니다.

## 13. 결론

마지막으로 네 가지를 강조합니다. Transformer 효율화는 보안성 자체가 아니고, NLP 공격은 별도 지표가 필요하며, prompt privacy는 leakage와 utility를 함께 봐야 하고, 수치는 실행 근거가 있을 때만 주장해야 합니다.
