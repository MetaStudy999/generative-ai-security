1. DP-SGD clipping/noise (W11): privacy guarantee와 membership inference 방어의 핵심

3. FedAvg aggregation (W10): federated poisoning/privacy 평가의 기준식
4. Prompt-injection ASR (W08): RAG/agent 보안 평가의 핵심 실패율
5. Scaled dot-product attention (W04/W07): Transformer/LLM 보안 해석의 기본 구조
6. Adversarial risk / robust accuracy (W03/W12): clean 성능과 robust 성능 분리
7. InfoNCE / masked reconstruction (W05): SSL pretraining 오염과 backdoor 위험 연결
8. DDPM forward process / GAN min-max (W06): 생성형 모델과 deepfake 탐지 지표 연결
9. MDP Bellman / DQN loss (W09): DRL 보상 조작과 검증 문제 연결
10. ECE / explanation stability (W15): 재현성·XAI·안전성 보고의 calibration 근거

## 7. PDF/DOCX 변환 시 수식 깨짐 점검 방법

1. Markdown 미리보기에서 `$$...$$`와 `\(...\)` 렌더링을 확인한다.
2. Pandoc 변환 시 HTML은 `--mathjax` 또는 `--katex` 옵션을 사용한다.
3. DOCX/PDF 변환 후 긴 수식이 표 셀에서 줄바꿈되는지 확인한다.
4. 표 안 수식의 `|` 문자는 Markdown 표 구분자로 깨질 수 있으므로 `\mid`, `\lVert`, `\rVert` 표기를 우선 사용한다.
5. 최종 PDF에서 한글과 LaTeX가 함께 깨지면 LaTeX/TeX Live와 `kotex` 기반 변환을 검토한다.

## 8. 후속 작업 제안

1. 각 원문 PDF에서 실제 수식/알고리즘 번호와 페이지를 확인해 `원문 위치`를 확정한다.
2. SUBSTITUTE 또는 서지 불일치 항목은 지정 논문 원문을 확보한 뒤 요약과 감사표를 다시 갱신한다.
3. 기말논문에는 Top 10 수식 중 DP-SGD, FedAvg, prompt-injection ASR, ECE를 우선 반영한다.
4. DOCX/PDF 제출 전 수식 렌더링과 표 셀 줄바꿈을 별도 체크리스트로 점검한다.
